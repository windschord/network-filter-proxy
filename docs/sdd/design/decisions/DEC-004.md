# DEC-004: Management API のバインドアドレスを環境変数で設定可能にする

## ステータス

**ステータス**: 承認済
**決定日**: 2026-03-12

## コンテキスト

Docker Compose 環境で ClaudeWork コンテナから Management API にアクセスする必要がある。現状は `127.0.0.1` 固定バインドのため、別コンテナからのアクセスが不可能。

### 現状の課題
- Management API が `127.0.0.1` に固定バインドされており、Docker network 経由の別コンテナからアクセスできない
- ClaudeWork がルール登録やヘルスチェックを行うには、Management API に到達可能である必要がある

---

## 検討した選択肢

### 選択肢A: `API_BIND_ADDR` 環境変数の追加（採用）

**概要**: 新しい環境変数 `API_BIND_ADDR` を追加し、バインドアドレスを設定可能にする。デフォルトは `127.0.0.1`。

**メリット**:
- 後方互換性を完全に維持（デフォルトが現行動作と同一）
- 既存の `API_PORT` との組み合わせで柔軟な設定が可能
- 設定の粒度が適切（アドレスとポートを独立に設定可能）

**デメリット**:
- `0.0.0.0` 設定時のセキュリティはインフラ側のネットワーク分離に依存

### 選択肢B: `API_ADDR` 環境変数で `host:port` を一括指定

**メリット**:
- 環境変数が 1 つで済む

**デメリット**:
- 既存の `API_PORT` との整合性が悪い（重複・競合の可能性）
- 破壊的変更となる

---

## 決定

**選択**: 選択肢A（`API_BIND_ADDR` 環境変数の追加）

**根拠**: 後方互換性を維持しつつ、最小限の変更で Docker Compose 環境をサポートできる。不正な IP アドレスが設定された場合は `127.0.0.1` にフォールバックすることで安全側に倒す。
