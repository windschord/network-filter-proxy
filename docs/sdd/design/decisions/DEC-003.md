# DEC-003: ルールの永続化を行わない（メモリのみ）

## ステータス

**ステータス**: 承認済
**決定日**: 2026-03-09

## コンテキスト

プロキシ再起動時にルールセットをどう扱うか。ファイル・DB への永続化を行うか、メモリのみで管理するか。

---

## 検討した選択肢

### 選択肢A: メモリのみ（永続化なし）（採用）

**メリット**:
- 実装がシンプル
- 外部依存（DB・ファイルシステム）なし
- Docker コンテナとしての運用が容易（ステートレス）

**デメリット**:
- 再起動時にルールが消える

### 選択肢B: ファイル（JSON）に永続化

**メリット**:
- 再起動時にルールが復元される

**デメリット**:
- ClaudeWork が再登録するため不要な複雑性
- ファイル I/O の競合制御が必要

---

## 決定

**選択**: 選択肢A（メモリのみ）

**根拠**: ClaudeWork がプロキシ起動後にルールを登録するフローが確立されており（ClaudeWork 連携フロー 1〜2）、永続化は不要。スコープ外として要件に明記されている。
