# DEC-002: HTTP ルーティングに Go 1.22 標準 net/http を使用

## ステータス

**ステータス**: 承認済
**決定日**: 2026-03-09

## コンテキスト

Management API のルーティング実装において、`gorilla/mux` 等のサードパーティルーターを使うか、Go 1.22 以降で強化された標準 `net/http` の ServeMux を使うかの選択。

---

## 検討した選択肢

### 選択肢A: 標準 net/http ServeMux（Go 1.22+）（採用）

**メリット**:
- Go 1.22 から `{param}` 形式のパスパラメータと HTTP メソッドマッチングをサポート
- 外部依存ゼロ
- `r.PathValue("sourceIP")` で簡潔にパラメータ取得

**デメリット**:
- Go 1.22 未満では使用不可（本プロジェクトは 1.26 のため問題なし）

### 選択肢B: gorilla/mux

**メリット**:
- 豊富なルーティング機能

**デメリット**:
- 追加依存
- Go 1.22 以降では標準で代替可能

---

## 決定

**選択**: 選択肢A（標準 net/http）

**根拠**: Go 1.26 を使用するため Go 1.22 以降の拡張 ServeMux を活用でき、外部依存を増やす必要がない。エンドポイント数も少なく（5 本）、標準ライブラリで十分。

## 影響

### 影響を受けるコンポーネント
- [api-handler](../components/api-handler.md) @../components/api-handler.md: `mux.HandleFunc("METHOD /path/{param}", handler)` 形式で登録
