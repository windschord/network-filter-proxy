# DEC-001: フォワードプロキシライブラリとして elazarl/goproxy を採用

## ステータス

**ステータス**: 承認済
**決定日**: 2026-03-09

## コンテキスト

Go で HTTP CONNECT をサポートするフォワードプロキシを実装する際、HTTP CONNECT のハンドリングを標準ライブラリ (`net/http`) のみで実装するか、専用ライブラリを使用するかの選択が必要。

### 現状の課題
- HTTP CONNECT トンネルの確立は標準ライブラリでは低レベルな実装が必要
- TLS パススルーの正確な実装にはバイナリパイプが必要

---

## 検討した選択肢

### 選択肢A: elazarl/goproxy（採用）

**概要**: Go 向けのフォワードプロキシライブラリ。CONNECT ハンドラーの登録 API を提供する。

**メリット**:
- HTTP CONNECT 処理が既実装
- ハンドラー登録 API がシンプル
- Go エコシステムで実績のあるライブラリ

**デメリット**:
- 外部依存が増える
- ライブラリの将来的なメンテナンス状況に依存

### 選択肢B: net/http のみでスクラッチ実装

**メリット**:
- 外部依存なし
- 完全なコントロール

**デメリット**:
- HTTP CONNECT トンネル確立・バイパイプの実装コストが高い
- バグのリスクが高い

---

## 決定

**選択**: 選択肢A（elazarl/goproxy）

**根拠**: 要件で明示的に指定されており、実装コストと信頼性のバランスが最適。
