# 性能要件

## 概要

フォワードプロキシとして、クライアントの通信に対してオーバーヘッドを最小限に抑えることを目標とする。

## 要件一覧

### NFR-PERF-001: プロキシ内部処理レイテンシ

**EARS記法**: システムは CONNECT リクエストに対して、ルール照合・許可判断などのプロキシ内部処理を 10ms 以内に完了しなければならない

**測定基準**:
- 指標: ルール照合と許可判断の内部処理時間（送信元 IP 取得・RuleStore 検索・Matcher 照合・レスポンス生成までの時間。外部ネットワーク遅延は含まない）
- 目標値: 10ms 以内
- 測定方法: ローカルのモック宛先（`127.0.0.1` 上の HTTP サーバー）を使用した負荷テストツール（`hey` 等）によるレイテンシ計測

**前提条件**: 宛先サーバーはローカルのモックエンドポイント（`127.0.0.1` 上）とし、外部ネットワーク遅延を排除した状態で計測する

**関連する機能要件**: REQ-001-001, REQ-002-001, REQ-003-001

**優先度**: 高

**検証方法**:
- [ ] ローカル環境でモック宛先を使用した 100 同時接続のベンチマークを実施し、P99 レイテンシが 10ms 以内であること

---

### NFR-PERF-002: 同時接続数

**EARS記法**: システムは 100 同時接続を処理している間、性能劣化なしに各接続のスループットを維持しなければならない

**測定基準**:
- 指標: 同時接続数
- 目標値: 100 同時接続
- 測定方法: 負荷テストによる同時接続シミュレーション

**関連する機能要件**: REQ-001-001

**優先度**: 中

**検証方法**:
- [ ] 100 同時 CONNECT 接続下でのスループット計測

---

### NFR-PERF-003: ルール照合計算量

**EARS記法**: システムはルール照合を O(n) 以下の計算量で実行しなければならない（n = 登録ルールエントリ数）

**測定基準**:
- 指標: ルールエントリ数に対する照合時間の線形性
- 目標値: 1000 エントリ時でも 1ms 以内
- 測定方法: ユニットテストでのベンチマーク

**関連する機能要件**: REQ-002-001

**優先度**: 中

**検証方法**:
- [ ] Go ベンチマークテストで 1000 エントリ照合時間を計測
