# 保守性要件

## 概要

運用・デバッグ・機能追加が容易な実装品質を維持することを目標とする。

## 要件一覧

### NFR-MNT-001: テストカバレッジ

**EARS記法**: システムはユニットテストカバレッジ 80% 以上を維持しなければならない

**測定基準**:
- 指標: `go test -cover` によるカバレッジ率
- 目標値: 80% 以上
- 測定方法: CI でのカバレッジ計測

**関連する機能要件**: 全 REQ

**優先度**: 高

**検証方法**:
- [ ] `go test -cover ./...` でカバレッジが 80% 以上であること

---

### NFR-MNT-002: 静的解析

**EARS記法**: システムのコードは `golangci-lint` の標準ルールセットに合格しなければならない

**測定基準**:
- 指標: lint エラー数
- 目標値: 0 件
- 測定方法: CI での `golangci-lint run` 実行

**優先度**: 中

**検証方法**:
- [ ] CI で `golangci-lint run` がエラーなしで完了すること

---

### NFR-MNT-003: Docker イメージサイズ

**EARS記法**: システムの Docker イメージは 30MB 以下でなければならない

**測定基準**:
- 指標: `docker image inspect` でのイメージサイズ
- 目標値: 30MB 以下
- 測定方法: CI でのイメージビルド後のサイズ確認

**優先度**: 低

**検証方法**:
- [ ] `gcr.io/distroless/static` ベースで静的バイナリをビルドし、イメージサイズが 30MB 以下であること

---

### NFR-MNT-004: Graceful shutdown の確実性

**EARS記法**: システムは SIGTERM 受信後 30 秒（またはSHUTDOWN_TIMEOUT 秒）以内に必ず終了しなければならない

**測定基準**:
- 指標: SIGTERM 送信から終了までの時間
- 目標値: SHUTDOWN_TIMEOUT 秒以内
- 測定方法: 統合テストで SIGTERM 送信後の終了時間を計測

**関連する機能要件**: REQ-007-001〜005

**優先度**: 高

**検証方法**:
- [ ] 統合テストで SIGTERM 後に指定タイムアウト以内でプロセスが終了することを確認
