# US-003: 送信元 IP ベースの環境識別とデフォルト拒否

## ユーザーストーリー

**私は** ClaudeWork オペレーターとして
**〜したい** 各 ClaudeCode コンテナの送信元 IP ごとに異なる許可ルールセットを適用したい
**なぜなら** コンテナ毎に必要な通信先が異なり、最小権限の原則を実現する必要があるから

## 受入基準（EARS記法）

### 機能要件

- **REQ-003-001**: プロキシがリクエストを受信した時、システムは接続元の IP アドレスを取得しなければならない
- **REQ-003-002**: 接続元 IP に対応するルールセットが登録されている場合、システムはそのルールセットを用いてアクセス判定を行わなければならない
- **REQ-003-003**: もし接続元 IP に対応するルールセットが登録されていない場合、システムは全ての通信を拒否し `403 Forbidden` を返しレスポンスヘッダーに `X-Filter-Reason: no-rules` を付与しなければならない
- **REQ-003-004**: 1 つの送信元 IP に対して、複数のルールエントリ（ドメイン + ポートの組み合わせ）を設定できなければならない

## データ要件

### ルールセット構造

```
送信元 IP → ルールセット {
  entries: [
    { host: "api.anthropic.com", port: 443 },
    { host: "*.github.com", port: 443 },
    ...
  ]
}
```

## 備考

- デフォルト拒否（deny-by-default）: 未登録 IP からの全通信は拒否
- ルールセットはメモリ上に保持し、API 経由で動的に変更可能
