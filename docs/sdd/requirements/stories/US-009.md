# US-009: ヘルスチェックサブコマンド

## ユーザーストーリー

**私は** ClaudeWork オペレーターとして
**〜したい** Docker Compose の `depends_on: condition: service_healthy` でプロキシコンテナの起動完了を待機したい
**なぜなら** ClaudeWork コンテナがプロキシ起動前にネットワークアクセスを試みると失敗するため、ヘルスチェックによる起動順序制御が必要だから

## 受入基準（EARS記法）

### 機能要件

- **REQ-009-001**: コマンドライン引数に `healthcheck` が指定された時、システムはプロキシモードではなくヘルスチェックモードで動作しなければならない
- **REQ-009-002**: ヘルスチェックモードの時、システムは `http://127.0.0.1:{API_PORT}/api/v1/health` に HTTP GET リクエストを送信しなければならない
- **REQ-009-003**: ヘルスチェックの HTTP レスポンスが 200 の場合、システムは終了コード 0 で終了しなければならない
- **REQ-009-004**: ヘルスチェックの HTTP リクエストが失敗した場合（接続エラー、タイムアウト、非 200 レスポンス）、システムは終了コード 1 で終了しなければならない
- **REQ-009-005**: ヘルスチェックモードの HTTP リクエストは 5 秒以内にタイムアウトしなければならない
- **REQ-009-006**: Dockerfile に `HEALTHCHECK` ディレクティブを定義し、`/filter-proxy healthcheck` を実行しなければならない

### Dockerfile 要件

- **REQ-009-007**: `HEALTHCHECK` ディレクティブの `--interval` は 15 秒としなければならない
- **REQ-009-008**: `HEALTHCHECK` ディレクティブの `--timeout` は 5 秒としなければならない
- **REQ-009-009**: `HEALTHCHECK` ディレクティブの `--retries` は 3 回としなければならない

## データ要件

### Dockerfile HEALTHCHECK ディレクティブ

```dockerfile
HEALTHCHECK --interval=15s --timeout=5s --retries=3 \
  CMD ["/filter-proxy", "healthcheck"]
```

### ヘルスチェックのフロー

```text
filter-proxy healthcheck
  |
  +-- HTTP GET http://127.0.0.1:{API_PORT}/api/v1/health
  |     |
  |     +-- 200 OK --> exit 0
  |     +-- それ以外 / エラー --> exit 1
  |
  +-- タイムアウト (5秒) --> exit 1
```

## 備考

- ヘルスチェックサブコマンドのアドレス解決: ワイルドカード（`0.0.0.0`, `::`）およびデフォルト（`127.0.0.1`）の場合は `127.0.0.1` に接続する。特定アドレス（`172.20.0.2`, `::1` 等）の場合はそのアドレスに接続する
- distroless ベースイメージには `curl`/`wget` がないため、Go バイナリ自身がヘルスチェッカーとして機能する
- US-005 の備考「Dockerfile 内で HEALTHCHECK 命令は使用しない」はこの要件により撤回される
