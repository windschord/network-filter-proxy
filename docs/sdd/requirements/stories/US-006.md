# US-006: 構造化ログ出力

## ユーザーストーリー

**私は** ClaudeWork オペレーターとして
**〜したい** プロキシが許可・拒否した通信とルール変更操作を構造化ログで記録したい
**なぜなら** セキュリティ監査・障害調査・通信の可視化のためにログが必要だから

## 受入基準（EARS記法）

### 機能要件

- **REQ-006-001**: 接続が許可された時、システムは送信元 IP・宛先ホスト・ポート・判定結果（`allow`）・タイムスタンプを含む JSON ログを出力しなければならない
- **REQ-006-002**: 接続が拒否された時、システムは送信元 IP・宛先ホスト・ポート・判定結果（`deny`）・拒否理由・タイムスタンプを含む JSON ログを出力しなければならない
- **REQ-006-003**: ルール変更操作（PUT / DELETE）が実行された時、システムは操作種別・対象 IP・タイムスタンプを含む JSON ログを出力しなければならない
- **REQ-006-004**: 環境変数 `LOG_FORMAT` が `text` の場合、システムはテキスト形式でログを出力しなければならない
- **REQ-006-005**: 環境変数 `LOG_FORMAT` が `json` または未設定の場合、システムは JSON 形式でログを出力しなければならない
- **REQ-006-006**: 環境変数 `LOG_LEVEL` に従い、システムは指定レベル以上のログのみを出力しなければならない

## データ要件

### 接続ログのフィールド

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `timestamp` | string (RFC3339) | 判定時刻 |
| `level` | string | ログレベル（`info` / `warn` 等）|
| `action` | string | `allow` または `deny` |
| `src_ip` | string | 送信元 IP アドレス |
| `dst_host` | string | 宛先ホスト名または IP |
| `dst_port` | int | 宛先ポート番号 |
| `reason` | string | 拒否時のみ（`no-rules` / `denied`） |

### ログレベル設定

| `LOG_LEVEL` 値 | 出力レベル |
|---------------|-----------|
| `debug` | debug 以上全て |
| `info` | info / warn / error |
| `warn` | warn / error |
| `error` | error のみ |
