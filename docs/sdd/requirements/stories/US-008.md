# US-008: Management API バインドアドレスの設定

## ユーザーストーリー

**私は** ClaudeWork オペレーターとして
**〜したい** Management API のバインドアドレスを環境変数で設定可能にしたい
**なぜなら** Docker Compose 環境で別コンテナ（ClaudeWork）から Management API にアクセスする必要があり、`127.0.0.1` 固定では Docker network 経由のアクセスができないから

## 受入基準（EARS記法）

### 機能要件

- **REQ-008-001**: 環境変数 `API_BIND_ADDR` が設定されている場合、システムは指定されたアドレスに Management API をバインドしなければならない
- **REQ-008-002**: 環境変数 `API_BIND_ADDR` が未設定の場合、システムは `127.0.0.1` にバインドしなければならない（後方互換性の維持）
- **REQ-008-003**: `API_BIND_ADDR` に不正な IP アドレス（例: `abc`、`999.999.999.999`）が指定された場合、システムは起動時にエラーログを出力し、`127.0.0.1` にフォールバックしなければならない
- **REQ-008-004**: システムは起動時に Management API の実際のバインドアドレスとポートをログに出力しなければならない

## データ要件

### 環境変数

| 変数 | デフォルト | 説明 |
|------|-----------|------|
| `API_BIND_ADDR` | `127.0.0.1` | Management API のバインドアドレス |

### 有効な値の例

| 値 | 説明 |
|-----|------|
| `127.0.0.1` | ローカルホストのみ（デフォルト） |
| `0.0.0.0` | 全インターフェース（Docker Compose 環境用） |
| `172.20.0.2` | 特定インターフェース |

## 備考

- `0.0.0.0` に設定した場合のセキュリティは Docker Compose のネットワーク分離に依存する。ドキュメントに「本番環境では適切なネットワーク分離を確保すること」を注記する
- 既存の `API_PORT` 環境変数との組み合わせで `{API_BIND_ADDR}:{API_PORT}` がリッスンアドレスとなる
