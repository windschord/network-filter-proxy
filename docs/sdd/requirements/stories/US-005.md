# US-005: ヘルスチェック

## ユーザーストーリー

**私は** ClaudeWork オペレーターとして
**〜したい** プロキシコンテナの正常性を外部から確認したい
**なぜなら** コンテナのヘルスチェックを Docker に組み込み、障害時に自動検知・再起動できるようにする必要があるから

## 受入基準（EARS記法）

### 機能要件

- **REQ-005-001**: `GET /api/v1/health` リクエストを受信した時、システムはプロキシプロセスと API プロセスの両方が正常な場合に HTTP 200 と稼働状態 JSON を返さなければならない
- **REQ-005-002**: ヘルスチェックレスポンスには `status`、`uptime_seconds`、`active_connections`、`rule_count` フィールドを含めなければならない
- **REQ-005-003**: システムが正常稼働中の場合、`status` フィールドは `"ok"` でなければならない

## データ要件

### GET /api/v1/health レスポンス 200

```json
{
  "status": "ok",
  "uptime_seconds": 3600,
  "active_connections": 5,
  "rule_count": 3
}
```

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `status` | string | 稼働状態（`"ok"`） |
| `uptime_seconds` | int | プロセス起動からの経過秒数 |
| `active_connections` | int | 現在の同時アクティブ接続数 |
| `rule_count` | int | 登録済み送信元 IP のルールセット数 |
