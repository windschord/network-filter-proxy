# US-001: HTTPS フォワードプロキシ

## ユーザーストーリー

**私は** ClaudeWork オペレーターとして
**〜したい** ClaudeCode コンテナの HTTPS 通信をプロキシ経由でフォワードしたい
**なぜなら** コンテナからのアウトバウンド通信を一元的に制御する基盤が必要だから

## 受入基準（EARS記法）

### 機能要件

- **REQ-001-001**: クライアントから HTTP CONNECT リクエストを受信した時、システムは TLS パススルーによるトンネルを確立しなければならない
- **REQ-001-002**: TLS トンネル確立中、システムは TLS 終端を行わず、クライアントとサーバー間の暗号化を維持しなければならない
- **REQ-001-003**: クライアントから GET / POST 等の通常 HTTP リクエストを受信した時、システムはそのリクエストを宛先サーバーへフォワードしなければならない
- **REQ-001-004**: システムは TCP ポート 3128 でプロキシリクエストをリッスンしなければならない
- **REQ-001-005**: 環境変数 `PROXY_PORT` が設定されている場合、システムは指定されたポートでリッスンしなければならない

## データ要件

- プロキシリスニングポート: デフォルト `3128`（環境変数 `PROXY_PORT` で変更可）
